# Operating Protocol

## 공통 업무 규칙

1. 각 에이전트는 자기 역할 밖의 결정을 확정하지 않는다.
2. 불확실한 정보는 `확인 필요`로 표시한다.
3. 다음 에이전트가 이어서 일할 수 있게 근거와 데이터를 남긴다.
4. 등록/게시/구매/결제/고객 연락은 사람 승인 전에는 하지 않는다.
5. 상표, 인증, 이미지 권리, 플랫폼 정책은 보수적으로 판단한다.

## 산출물 공통 형식

모든 에이전트는 산출물 마지막에 아래 블록을 남긴다.

```text
HANDOFF
- 다음 담당:
- 넘길 자료:
- 확인 필요:
- 하드 스톱:
- 추천 액션:
```

## 상태 값

- `TODO`: 아직 시작 전
- `IN_PROGRESS`: 진행 중
- `BLOCKED`: 외부 확인 필요
- `READY_FOR_REVIEW`: 다음 에이전트 또는 사람이 검토 가능
- `DONE`: 완료

## 결정 값

- `PROCEED`: 다음 단계 진행 가능
- `REVIEW`: 추가 확인 후 진행 가능
- `HOLD`: 지금은 보류
- `KILL`: 폐기 권장

## 파일 사용 규칙

- 업무 배정은 각 에이전트의 `inbox.md`에 기록한다.
- 누적 주의사항은 `memory.md`에 기록한다.
- 반복 가능한 양식은 `output_template.md`를 따른다.
