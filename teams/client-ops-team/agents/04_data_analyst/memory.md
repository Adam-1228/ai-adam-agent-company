# Memory - 최분석

## 추적하는 상태

- 활성 케이스의 KPI 측정 시작일 (김은보로부터 받음)
- 직전 4주 / 직전 12주 비교 기준 baseline (케이스별, 업종별)
- 정총괄로부터 받은 리포트 1차 검수 피드백 (재발 방지용)
- 표본 미달 케이스 목록 (결론 금지 플래그)
- 의료/법률/세무 영역 인접 표현 사전 (정총괄 합의)

## 저장 위치

- 원본 데이터: `reports/raw_*.jsonl` (gitignore, 박실행/이용대로부터)
- 가공 데이터: `reports/processed_*.parquet` (gitignore)
- 리포트 초안: `reports/weekly_YYYY-Www.md`, `reports/weekly_YYYY-Www.pdf` (gitignore)
- 베이스라인: `reports/baseline_{industry}.json` (gitignore)
- 분석 스크립트 (Codex 산출): `scripts/analysis/` (이건 공개 가능 — 비밀 없음)

## 갱신 주기

- 일일: 박실행/이용대 데이터 도착 시 자동 통합
- 주간: 월요일 10:00 — 직전 주 리포트 초안 생성
- 월간: 매월 1영업일 — 종합 리포트 + 베이스라인 재계산
- 사후 갱신: 정총괄 검수 피드백 도착 시 즉시 반영

## 다른 에이전트와 공유하는 메모리

- 박실행: 매주 월 09:00에 박실행이 익명 집계 JSONL 푸시 (정시 실행률, 전송 성공률, 실패 사유)
- 이용대: 매주 월 09:30에 응대 카테고리 집계 (P0/P1/P2/P3 건수, 카테고리 분포)
- 김은보: 신규 케이스 등록 시 KPI 측정 시작일·비교 기준 (전월/동종 업종)
- 정총괄: 모든 리포트 검수 큐 인입
- 커머스 팀: `shared/handoff_to_commerce.md` 형식의 익명 집계 패키지

## 절대 기억하면 안 되는 것

- 고객사 실명·실주소·실대표명
- 고객사 매출/원가/마진의 절대값 (지수화 또는 기준 대비 비율만)
- 응대 본문 (해시만, 본문 보관은 이용대 영역)
- 다른 고객사 식별 가능한 횡단 비교 (반드시 업종/주차 단위로 집계)

## 누적 주의사항

- 표본 < 20: "현상 보고", 결론 금지.
- "상승"/"감소"는 반드시 비교 기간 명시. ("지난주 대비", "직전 4주 평균 대비")
- "효과 있음"/"효과 없음" 표현 금지. "상관관계 관찰" 또는 "직접 인과 증거 부족".
- 리마인더와 노쇼 감소를 묶어 단정하지 않는다. A/B 또는 시점 차분 분석 명시.
- PDF 리포트의 차트 축은 0부터. 누적 비율 차트는 사용하지 않는다.
- 한 페이지 첫 줄은 항상 TL;DR. 표는 그 다음.
