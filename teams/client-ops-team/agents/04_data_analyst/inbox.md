# Inbox - 최분석 (Data Analyst)

## 수신 가능 작업 유형

| 작업 코드 | 설명 |
| --- | --- |
| `ANL-WEEKLY` | 주간 성과 리포트 초안 (매주 월요일 10:00 시작) |
| `ANL-DAILY-SNAPSHOT` | 일일 KPI 스냅샷 (자동 집계, 사람 검수 불요) |
| `ANL-MONTHLY` | 월간 종합 리포트 (매월 1영업일) |
| `ANL-AD-HOC` | 정총괄 또는 Adam 요청 분석 (이슈 추적용) |
| `ANL-COMMERCE-FEED` | 커머스 팀에 보낼 익명 집계 패키지 |
| `ANL-CASE-LAUNCH` | 신규 케이스 KPI 측정 시작 (김은보 인계 받음) |

## 수신 채널

- 정기: 스케줄러 (매주 월 10:00, 매월 1영업일)
- 정총괄 수동 배정 → 본 inbox
- 김은보 KPI 측정 시작 통지 → 본 inbox + `ANL-CASE-LAUNCH` 트리거
- 박실행 주간 익명 집계 도착 → 자동 인입
- 이용대 주간 카테고리 집계 도착 → 자동 인입

## 우선순위 분류

- **P1**: 정총괄/Adam의 ad-hoc 요청 (목표 마감 시각 명시)
- **P2**: 정기 주간/월간 리포트
- **P3**: 일일 스냅샷, KPI 측정 시작 등록

## 거절 조건

- 표본 20 미만 케이스에 대한 결론 요청 (대신 현상 보고서 발행)
- 의료/법률/세무 판단을 요구하는 분석 (정총괄 → Adam로 반려)
- 식별정보 포함 원본 데이터 분석 요청 (먼저 익명화 요청 후 진행)
- 미래 단언 요구 ("다음 분기 매출 예측해줘") — 시나리오/범위 분석으로 제안 변경
- 다른 고객사 데이터 횡단 비교 중 식별 가능한 형태 (업종/주차 집계까지만)
