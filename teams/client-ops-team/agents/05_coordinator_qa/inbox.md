# Inbox - 정총괄 (Coordinator / QA)

## 수신 가능 작업 유형

| 작업 코드 | 설명 | 출처 |
| --- | --- | --- |
| `QA-ONB-REVIEW` | 김은보 온보딩 패키지 검수 | 01 |
| `QA-OPS-DAILY` | 박실행 일일 실행 요약 검수 | 02 |
| `QA-OPS-FAILURE` | 박실행 즉시 실패/거절 알림 처리 | 02 |
| `QA-CS-P0` | 이용대 P0 에스컬레이션 카드 → Adam 통지 | 03 |
| `QA-CS-DRAFT` | 이용대 P1/P2 응대 초안 검수 | 03 |
| `QA-CS-POST-REVIEW` | 이용대 P3 자동응답 사후 검수 | 03 |
| `QA-ANL-WEEKLY` | 최분석 주간 리포트 검수 → Adam 보고 | 04 |
| `QA-ANL-COMMERCE-FEED` | 최분석 커머스 송부 익명 집계 검수 | 04 |
| `QA-ADAM-ESCALATION` | 6대 케이스 발생 시 Adam 직접 통지 | 본인 발 |
| `QA-CONFLICT` | 4명 간 충돌 조정 | 본인 발 |

## 수신 채널

- 자동 인입: 4명의 산출물이 정해진 시점에 본 inbox로 큐잉
- Telegram: P0 / 박실행 crit / 6대 케이스 즉시 알림
- Adam 수동 배정: 본 inbox

## 우선순위 분류 (검수 SLA)

| 우선순위 | 케이스 | SLA |
| --- | --- | --- |
| **P0** | 이용대 P0 에스컬레이션, 박실행 crit, 6대 케이스 | 즉시 (5분 이내 Adam 통지) |
| **P1** | 박실행 일일 요약, 이용대 P1 초안 | 1시간 이내 |
| **P2** | 이용대 P2 초안, 김은보 온보딩 패키지 | 4시간 이내 |
| **P3** | 최분석 주간 리포트, 이용대 P3 사후 검수 | 영업일 내 |

## 거절 조건 (정총괄도 거절한다)

- 자기 자신이 작성한 산출물 검수 요청 (Adam에게)
- HANDOFF 블록 부재 산출물 (자동 반려)
- 출처/비교기준이 0개인 숫자 포함 리포트 (반려)
- 식별정보가 마스킹되지 않은 외부 송부용 패키지 (반려)
- 의료/법률/세무 단정 표현 포함 산출물 (반려)

## 현재 큐

아직 배정된 작업 없음.

(샘플 결정문은 `tasks/done.md`의 QA 항목 참고)
