# Inbox - 김은보 (Onboarding Manager)

## 수신 가능 작업 유형

| 작업 코드 | 설명 |
| --- | --- |
| `ONB-INTAKE` | 신규 고객 1차 정보 수집 (상호, 업종, 담당자, 연락처) |
| `ONB-CONTRACT` | 계약 조건 검증 (요금제/주기/기간/특약) |
| `ONB-FORBIDDEN` | 우리가 받지 말아야 할 업무 식별 (의료/법률/세무 등) |
| `ONB-PERMSCAN` | 외부 시스템 API 권한 감사 |
| `ONB-PACKAGE` | 온보딩 패키지(요약+위험신호+30일 운영계획) 작성 |
| `ONB-HANDOFF-OPS` | 박실행에게 정기 작업 등록 인계 |
| `ONB-REVIEW-REQUEST` | 정총괄에게 셋업 검수 요청 |

## 수신 채널

- 운영 담당자 수기 입력 → 본 `inbox.md`
- Adam 직접 배정 → 본 `inbox.md`
- 정총괄로부터 재작업 요청 → 본 `inbox.md` (이유 포함)

자동 채널(Slack/메일)은 v0.1에서 미사용. 모든 신규 고객은 사람이 명시적으로 큐에 올린다.

## 우선순위 분류

- **P1**: 계약 체결 완료, 7일 내 운영 시작 예정
- **P2**: 계약 체결 완료, 14일 이내 운영 시작
- **P3**: 계약 검토 중 또는 가망 단계

같은 우선순위라면 계약 체결일이 빠른 순.

## 거절 조건

다음 케이스는 인박스에 받지 않고 즉시 정총괄로 반려한다.

- 사업자번호 미제공 상태에서 운영 시작 요청
- 의료 행위, 변호사 자문, 세무 신고 대행 등 우리 업무 범위 밖
- 결제 정보 출처가 본인 명의가 아님
- "성과 보장", "환불 보장" 등 사전 약속이 계약서에 명시되어 있음
- 고객측 요구 권한이 운영에 필요한 최소 범위를 초과 (예: DB 풀 쓰기)
