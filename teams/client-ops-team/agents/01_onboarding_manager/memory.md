# Memory - 김은보

## 추적하는 상태

- 진행 중 온보딩 케이스 ID 목록 (상태: 정보수집 / 검증 / 권한감사 / 패키지작성 / 셋업검수 / 완료)
- 각 케이스의 "확인 필요" 항목 잔량 (0이 되어야 다음 단계 진행)
- 정총괄 1차 검수 통과율 (월별)
- 30일 내 이탈한 고객의 공통 신호 (사후 학습용, 식별정보 없이)

## 저장 위치

- 상태 머신: `tasks/active.md` 내 `ONB-*` 항목
- 완료 케이스 요약: `tasks/done.md`
- 케이스별 상세 패키지: `reports/` (gitignore, 로컬/EC2 전용)
- 누적 주의사항(이 파일): 패턴화된 학습만 (개인정보 없음)

## 갱신 주기

- 인박스 처리 시점 (실시간)
- 정총괄 검수 통과/반려 시점 (실시간)
- 매주 금요일 16:00 — 그 주에 새로 학습한 패턴을 이 파일에 1줄로 추가

## 다른 에이전트와 공유하는 메모리

- 박실행: `shared/handoff_contract.md`의 `ONB → OPS` 섹션에 정의된 정기 작업 명세
- 이용대: 고객별 응대 톤·금지 키워드 카드 (이용대 `memory.md`로 전달)
- 최분석: KPI 측정 시작일과 비교 기준
- 정총괄: 셋업 검수 시 본 케이스의 위험 신호 요약

## 절대 기억하면 안 되는 것

- 사업자번호 전체 (마스킹된 뒤 4자리만 허용)
- 주민등록번호, 카드번호, 의료기록, 진단명
- 고객사 내부 매출/수익 절대값 (필요시 정총괄 경유로 최분석에게만)
- 계약서 본문 (저장 위치 포인터만, 본문은 외부 보관)
- 담당자 개인 연락처 풀텍스트 (`010-XXXX-1234` 형태로 마스킹)

## 누적 주의사항

- 미용 시술 업종은 "일반 관리"와 "의료 행위"가 섞이는 경우가 잦다. 시술 메뉴 리스트를 받기 전에 운영 시작하지 않는다.
- "성과 보장", "환불 100%" 같은 표현이 영업 단계 메모에 남아 있으면 정총괄을 반드시 거친다.
- 신규 고객이 첫 주에 권한 추가 요청을 하면 자동 승인하지 않는다. 사유와 데이터 흐름을 다시 본다.
- 사업자번호와 대표자명은 외부 API로 1회 검증한 결과를 캡처로만 남기고, 본 메모리에는 검증일자/결과 라벨만 남긴다.
